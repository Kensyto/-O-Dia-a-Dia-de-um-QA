# O Dia a Dia de um QA

Este repositório contém a documentação gerada para o desafio de projeto "O Dia a Dia de um QA", focado em processos de testes manuais em ambientes ágeis.

## Objetivo do Projeto
O objetivo foi compreender e praticar atividades relativas ao dia a dia de um QA, incluindo levantamento de requisitos, análise de histórias de usuário, planejamento de testes e documentação de cenários e casos de teste.

## Conteúdo do Repositório

O projeto foca na funcionalidade de **Pagamentos via Pix** e inclui os seguintes documentos em PDF:

1.  **[workflow_bug_lifecycle.pdf](./workflow_bug_lifecycle.pdf)**: Define o fluxo de trabalho ágil e o ciclo de vida de um bug no projeto.
2.  **[user_stories.pdf](./user_stories.pdf)**: Apresenta duas User Stories detalhadas para pagamentos Pix, seguindo os princípios INVEST e critérios de aceite.
3.  **[mind_map.pdf](./mind_map.pdf)**: Mapa mental estruturado para a funcionalidade de pagamento via chave Pix.
4.  **[test_cases_step_by_step.pdf](./test_cases_step_by_step.pdf)**: Dois casos de teste detalhados utilizando a técnica passo a passo.
5.  **[test_cases_bdd.pdf](./test_cases_bdd.pdf)**: Dois casos de teste escritos em formato BDD (Gherkin).
6.  **[test_plan_cycle.pdf](./test_plan_cycle.pdf)**: Documentação de um Plano de Teste e um Ciclo de Teste para o projeto.

## Tecnologias e Ferramentas Utilizadas
- **Metodologia Ágil (Scrum/Kanban)**
- **User Stories (INVEST)**
- **BDD (Behavior Driven Development)**
- **Mind Mapping**
- **Test Management Concepts**
